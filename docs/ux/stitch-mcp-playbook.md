# Stitch MCP Playbook

Use this file only if the project actually uses Stitch.

## Recommended Usage

- ideation for early layout direction
- draft screen exploration
- approved snapshot reference when the team explicitly accepts it

## Guardrails

- Do not treat raw Stitch output as the sole final implementation source.
- Record project ID, screen references, and approval context when Stitch output
  becomes part of task evidence.
- Keep final implementation tied to explicit acceptance evidence.
