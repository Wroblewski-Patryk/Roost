# UX/UI MCP Collaboration

Use MCP design context deliberately for UX-heavy tasks.

## Source-Of-Truth Policy

- Primary source: Figma MCP when a design source exists.
- Secondary source: Stitch MCP for ideation, draft exploration, or approved
  snapshot workflows.
- Stitch-only output must not be the sole final implementation source unless
  the project explicitly records an exception.

## Required Evidence For UX Tasks

- source reference
- required states: `loading`, `empty`, `error`, `success`
- responsive verification: `desktop`, `tablet`, `mobile`
- accessibility verification
- parity or screenshot evidence when relevant

## Task Contract

Record UX evidence in `.codex/templates/task-template.md` fields or equivalent
task notes.
